# 音量控制功能说明

## 功能概述

实现了当收到音频包或开始录音时自动降低音乐音量的功能，确保音频内容不会被背景音乐干扰。

## 实现的功能

### 1. 音频包接收时的音量控制
- **触发条件**: 当收到UDP音频包时
- **动作**: 将音乐音量降低到 `min(当前音量, 40)`
- **恢复条件**: 2秒内没有收到新的音频包时自动恢复

### 2. 录音时的音量控制
- **触发条件**: 当触发唤醒词并开始录音时
- **动作**: 将音乐音量降低到 `min(当前音量, 40)`
- **恢复条件**: 录音结束时自动恢复

### 3. 智能音量管理
- **防重复降低**: 只在第一次触发时降低音量，避免重复操作
- **状态协调**: 当同时有音频包接收和录音时，优先保持低音量
- **安全恢复**: 只有在所有触发条件都结束时才恢复音量

## 代码实现

### 新增的实例变量

```python
# 音频包接收状态管理（用于音量控制）
self.audio_packet_receiving = False
self.original_music_volume = None
self.volume_reduced = False
self.last_audio_packet_time = 0
self.volume_restore_timer = None

# 录音状态管理（用于音量控制）
self.recording_volume_reduced = False
```

### 核心方法

#### 1. `_reduce_music_volume(reason)`
通用的音量降低方法，支持不同的触发原因。

#### 2. `_reduce_music_volume_for_audio_packet()`
专门用于音频包接收时的音量降低。

#### 3. `_reduce_music_volume_for_recording()`
专门用于录音时的音量降低。

#### 4. `_restore_music_volume(reason)`
音量恢复方法，支持不同的恢复原因。

#### 5. `_schedule_volume_restore(reason)`
延迟音量恢复，使用定时器在2秒后执行。

### 集成点

#### 1. 音频包接收检测
在 `_audio_playback_worker` 方法中：
- 接收到音频包时调用 `_reduce_music_volume_for_audio_packet()`
- 超时处理中检查是否需要恢复音量

#### 2. 录音开始/结束
- `start_recording()`: 调用 `_reduce_music_volume_for_recording()`
- `stop_recording()`: 检查条件后恢复音量

#### 3. 资源清理
在 `cleanup()` 方法中清理定时器和重置状态。

## 使用场景

### 场景1: 音频包接收
```
音乐播放中 → 收到音频包 → 音量降低到40 → 音频包停止2秒后 → 音量恢复
```

### 场景2: 录音
```
音乐播放中 → 触发唤醒词开始录音 → 音量降低到40 → 录音结束 → 音量恢复
```

### 场景3: 同时进行
```
音乐播放中 → 开始录音(音量降低) → 收到音频包 → 录音结束(音量保持低) → 音频包停止2秒后 → 音量恢复
```

## 特性

1. **智能阈值**: 只有当前音量大于40时才降低，避免音量过低
2. **状态管理**: 防止重复降低音量，确保状态一致性
3. **延迟恢复**: 音频包停止后延迟2秒恢复，避免频繁切换
4. **条件检查**: 恢复音量前检查是否还有其他活动需要保持低音量
5. **异常处理**: 完整的错误处理和日志记录
6. **资源管理**: 正确的定时器清理和状态重置
